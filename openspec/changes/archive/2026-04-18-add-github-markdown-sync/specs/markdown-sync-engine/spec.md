## ADDED Requirements

### Requirement: GitHub文件拉取
系统 SHALL 支持从GitHub仓库拉取指定目录下的所有.md文件，支持公开和私有仓库。

#### Scenario: 拉取公开仓库文件
- **WHEN** 用户触发同步公开仓库，无需配置令牌
- **THEN** 系统拉取指定分支和目录下的所有.md文件

#### Scenario: 拉取私有仓库文件
- **WHEN** 用户触发同步私有仓库，配置了有效令牌
- **THEN** 系统使用令牌认证后拉取所有.md文件

#### Scenario: 拉取失败处理
- **WHEN** 拉取过程中出现网络错误、权限不足等问题
- **THEN** 系统记录错误信息，终止同步，返回失败状态

### Requirement: 同步去重机制
系统 SHALL 实现双重去重机制，避免重复导入相同文档。

#### Scenario: 新文件同步
- **WHEN** 拉取的文件路径在当前仓库中没有同步记录
- **THEN** 系统解析文件内容，创建新文章，记录映射关系

#### Scenario: 已存在文件内容无变化
- **WHEN** 拉取的文件路径已存在，且内容哈希值与上次同步相同
- **THEN** 系统跳过该文件，不做任何修改

#### Scenario: 已存在文件内容有变化
- **WHEN** 拉取的文件路径已存在，但内容哈希值与上次不同
- **THEN** 系统更新对应文章的内容和元数据，更新同步记录

### Requirement: 异步同步执行
系统 SHALL 支持异步同步操作，避免长时间阻塞前端请求。

#### Scenario: 触发同步
- **WHEN** 用户点击"开始同步"按钮
- **THEN** 系统立即返回同步任务ID，后台异步执行同步操作

#### Scenario: 查询同步状态
- **WHEN** 用户使用任务ID查询同步状态
- **THEN** 系统返回当前同步进度、已处理文件数、成功/失败数量等信息

### Requirement: 事务一致性
系统 SHALL 保证同步操作的事务一致性，同步失败时自动回滚。

#### Scenario: 部分文件同步失败
- **WHEN** 同步过程中部分文件处理失败
- **THEN** 已成功处理的文件保留，失败的文件记录错误信息，不影响整体同步

#### Scenario: 同步过程中断
- **WHEN** 同步过程中系统异常中断
- **THEN** 下次同步时可从中断处继续，或选择重新同步所有文件
