## ADDED Requirements

### Requirement: GitHub仓库配置管理
系统 SHALL 支持对GitHub仓库配置的增删改查操作，每个配置包含仓库地址、分支、文档根目录、访问令牌、默认分类、默认标签等字段。

#### Scenario: 创建仓库配置
- **WHEN** 用户在后台填写完整的GitHub仓库信息并提交
- **THEN** 系统验证信息有效性，加密存储访问令牌，保存配置到数据库

#### Scenario: 修改仓库配置
- **WHEN** 用户修改已存在的仓库配置并提交
- **THEN** 系统验证新信息有效性，更新数据库记录，令牌字段若为空则保留原有值

#### Scenario: 删除仓库配置
- **WHEN** 用户删除已存在的仓库配置
- **THEN** 系统删除配置记录，同时删除关联的同步记录和映射关系，但保留已同步的文章

#### Scenario: 查看仓库配置列表
- **WHEN** 用户访问仓库配置列表页面
- **THEN** 系统返回所有配置的基本信息，不包含访问令牌明文

#### Scenario: 验证仓库连通性
- **WHEN** 用户点击"测试连接"按钮
- **THEN** 系统使用配置的令牌尝试访问GitHub仓库，返回连通性测试结果

### Requirement: 访问令牌安全存储
系统 SHALL 采用AES加密算法存储GitHub访问令牌，数据库中不保存明文令牌。

#### Scenario: 令牌加密存储
- **WHEN** 用户保存包含访问令牌的配置
- **THEN** 系统使用系统密钥加密令牌后存储到数据库

#### Scenario: 令牌解密使用
- **WHEN** 系统需要调用GitHub API时
- **THEN** 系统从数据库读取加密令牌，解密后使用，使用后立即从内存中清除
