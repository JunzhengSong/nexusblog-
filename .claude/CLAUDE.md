# 项目开发规则
本文件定义了项目开发过程中必须遵守的规则：

## 1. 全程使用中文交流
所有和用户的所有对话、回复、功能说明、错误提示、代码注释（如需添加的注释内容全部使用中文，避免使用英文。

## 2. 开发完功能后自动重启/热更新项目
每次功能开发完成、代码修改并测试通过后，自动执行项目重启或触发热更新，保证浏览器访问的网页和最新代码保持同步，避免用户可以直接预览最新功能无需手动重启服务。

## 3. TDD 测试驱动开发
所有功能开发必须遵守测试驱动开发流程：
- 先编写对应的单元测试/集成测试用例
- 再编写功能代码实现需求
- 必须保证所有测试用例全部通过才能交付功能
- 不允许出现没有测试覆盖的功能上线
